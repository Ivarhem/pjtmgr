# 02. Critical Questions — 설계 결정 질문

## 목표

spec 초안에서 **답변에 따라 설계가 달라지는 핵심 질문**만 추출한다.
사소한 질문은 제외하고, 프로젝트 방향을 결정하는 질문에 집중한다.

## 입력

- `docs/SPEC.md` 초안 (01단계 산출물)

## 출력

3~7개의 핵심 질문. 각 질문에 다음을 포함한다.

### 질문 형식

```
## Q1. [질문]

왜 중요한가:
[이 질문의 답에 따라 어떤 설계 결정이 달라지는지]

선택지:
- A: [선택지와 그에 따른 영향]
- B: [선택지와 그에 따른 영향]
```

## 질문 포함 기준

다음 중 하나 이상에 해당하는 경우만 포함한다.

- 데이터 모델이 달라지는 경우
- UX 흐름이 달라지는 경우
- 운영 방식이 달라지는 경우
- 프로젝트 범위가 크게 바뀌는 경우
- 기술 스택 선택이 달라지는 경우

## 주의사항

- "이것도 추가하면 좋겠다"는 범위 확장 질문은 제외
- 구현 세부사항 질문은 제외 (아키텍처 단계에서 결정)
- 사용자가 이미 명시한 제약을 다시 묻지 않는다
