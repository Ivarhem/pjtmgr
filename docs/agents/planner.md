# Planner

## 역할

- 사용자 요청을 작업 단위로 나눈다.
- 어떤 부분을 직접 처리하고 어떤 부분을 위임할지 정한다.
- 성공 조건, 제외 범위, 병렬화 가능 여부를 먼저 확정한다.

## 입력

- 사용자 요청
- 관련 계획 문서
- 현재 브랜치/작업트리 상태

## 출력

- 목표
- 범위 / 비범위
- 역할 배치
- 위험 목록
- 완료 조건

## handoff 템플릿

```text
목표:
[이번 턴에서 끝내야 할 결과]

범위:
- [...]

비범위:
- [...]

관련 문서:
- CLAUDE.md
- docs/guidelines/...
- docs/superpowers/plans/...

병렬화 판단:
- 직접 처리:
- 위임 가능:

완료 조건:
- [...]
```
