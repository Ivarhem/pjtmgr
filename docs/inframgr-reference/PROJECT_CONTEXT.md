# Project Context

## 목적

대형 SI 프로젝트 수행 과정에서 분산된 Excel 원장 대신,
프로젝트 기술 인벤토리를 단일 원장으로 관리하는 사내 전용 웹앱을 구축한다.

핵심 목적은 다음과 같다.

- 프로젝트 단계 및 산출물 관리
- 기술 자산과 네트워크 정보의 정합성 확보
- 정책 기준과 적용 상태의 추적
- 업체 및 담당자 연락망의 중앙화
- 프로젝트 종료 후에도 기술 정보 보존

## 사용자

1. **프로젝트 수행 인력** — 프로젝트별 자산, IP, 포트맵, 담당자 정보 조회/수정
2. **기술 리더 / PMO 보조 인력** — 단계 및 산출물 현황, 정책 적용 상태 관리
3. **관리자** — 정책 정의, 기준 데이터, 사용자 및 권한 관리

## 핵심 문제

- 참여 인원별로 서로 다른 Excel 원장을 유지하여 정보가 불일치함
- 장비 / IP / 포트맵 정보가 최신 상태로 유지되지 않음
- 담당자 연락처를 즉시 찾기 어려움
- 프로젝트 종료 후 기술 정보가 산발적으로 소실됨

## 핵심 데이터

- **Project** — 기술 인벤토리의 상위 컨텍스트
- **ProjectPhase / Deliverable** — 단계 및 산출물
- **Asset** — 서버, 네트워크 장비, 보안 장비 등 기술 자산
- **AssetIP** — 자산에 속한 IP 정보
- **PortMap** — 자산 간 통신 관계
- **PolicyDefinition / PolicyAssignment** — 정책 정의와 적용 상태
- **Partner / Contact / AssetContact** — 업체, 담당자, 자산 연결

## 핵심 원칙

- 본 시스템은 PM Tool이 아니라 프로젝트 기술 인벤토리 시스템이다.
- 탐색 중심 엔티티는 `Asset`이다.
- 정책은 정의와 적용 상태를 반드시 분리한다.
- 태스크, 일정, 알림, 메시징, 협업 기능은 범위에서 제외한다.
- UI는 테이블 중심이며 빠른 조회와 정합성 확보를 우선한다.
- 외부 시스템 연동은 MVP 범위에서 제외한다.

## 기술 스택

- Backend: Python 3.11+ / FastAPI / SQLAlchemy
- DB: PostgreSQL
- Frontend: Jinja2 / HTMX / AG Grid Community / Vanilla JS
- 인증: 세션 기반

## 현재 단계

설계 및 MVP 구현 준비 단계.

우선순위는 다음과 같다.

1. 프로젝트 + 단계 + 산출물
2. Asset 인벤토리
3. IP 인벤토리
4. PortMap 인벤토리
5. 정책 관리
6. 업체 및 담당자 관리

## 향후 방향

- Excel Import/Export
- 감사 로그
- 정책 점검 리포트
- 역할 세분화
- 외부 시스템 연동
