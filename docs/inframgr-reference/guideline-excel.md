# Excel 연동 작업 지침

> Excel Import/Export 기능 작업 시 참조.

---

## 현재 상태

- Excel Import/Export는 MVP 범위에서 제외한다.
- 이 문서는 후속 단계에서 Excel 기능을 도입할 때의 기본 원칙을 정의한다.

## 기본 규칙

- 업로드는 지정된 템플릿 형식만 허용한다.
- 다운로드는 현재 조회 결과 또는 명시된 보고 포맷만 내보낸다.
- 날짜, IP, 포트, 상태값 등 핵심 필드는 서버에서 다시 검증한다.
- 오류 응답은 행 번호와 필드명을 포함해 사용자가 수정 가능한 수준으로 제공한다.
- 파일 업로드 시 확장자 및 MIME 타입을 검증한다 (`.xlsx`만 허용).
- 처리 라이브러리는 `openpyxl` 우선, 대량 데이터 변환이 필요하면 `pandas`를 보조적으로 사용한다.

## 후속 도입 우선순위

1. 프로젝트 / Asset 마스터 Import
2. IP 인벤토리 Import
3. PortMap Import
4. 정책 적용 현황 Import
5. 조회 결과 Export

## 주의사항

- 프로젝트 범위 IP 중복 정책과 Excel Import 정책은 동일해야 한다.
- 자산명, 정책코드, 업체명 등 자연키 후보는 Import 전에 정규화 규칙을 먼저 확정해야 한다.
- 부분 성공 허용 여부는 기능 도입 시 별도 결정하고 `docs/DECISIONS.md`에 기록한다.
