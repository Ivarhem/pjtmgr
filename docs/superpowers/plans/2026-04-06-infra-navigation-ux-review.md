# 인프라 메뉴 구조 / UI-UX 개선안

> 인프라모듈의 탐색 구조와 주요 작업 화면을 제품 목적 기준으로 재정렬하기 위한 참조 계획.
> 멀티에이전트로 진행할 때는 벤더 전용 sub-skill 이름 대신 `docs/guidelines/agent_workflow.md`와 필요한 `docs/agents/*.md`를 기준으로 역할을 나눈다.

## 목적

- 인프라모듈의 실제 주 작업축을 메뉴 구조에 반영한다.
- 전역 topbar 컨텍스트 의존도를 낮추고, 각 화면 본문에서도 현재 작업 기준을 즉시 이해할 수 있게 한다.
- 회계모듈 대비 약한 인프라 화면의 작업 밀도와 진입 흐름을 정리한다.

## 현재 문제 요약

1. 인프라 서브메뉴가 `프로젝트 / 자산 / 업체 / 이력` 4개로 너무 얕다.
2. `IP 인벤토리`, `포트맵`, `물리배치`, `역할 기준 보기`가 자산 화면 내부 탭에 숨어 있다.
3. `거래처 관리 / 카탈로그 / 로그 / 시스템 관리`가 핵심 업무 메뉴와 같은 위계에 놓여 있다.
4. `업체` 화면은 topbar 선택기에 의존하고, 본문에서 현재 고객사/프로젝트 컨텍스트가 약하다.
5. 인프라 화면 전반이 "지금 무엇을 할 수 있는지"를 회계 화면만큼 명확히 드러내지 못한다.

## 목표 사용자 관점

- 인프라 사용자는 자산 목록만 보는 사람이 아니다.
- 실제 시작점은 `프로젝트`, `IP 인벤토리`, `포트맵`, `배치`, `업체 연결` 중 하나일 수 있다.
- 따라서 메뉴는 데이터 모델 중심보다 작업 시작점 중심으로 보여야 한다.

## 제안 1. 메뉴 정보 구조 재편

### 원칙

- 1차 메뉴는 "주 작업 진입점"만 둔다.
- 기준정보/관리성 화면은 별도 묶음으로 분리한다.
- 자산 화면 내부 탭은 세부 보기 전환용으로만 남기고, 독립 업무 화면은 전역 진입점을 제공한다.

### 제안 메뉴

#### 인프라 작업 메뉴

- `프로젝트`
  - 프로젝트 목록, pin 프로젝트, 기본 정보, 단계/산출물
- `자산`
  - 자산 목록, 역할 기준 보기
- `네트워크`
  - IP 인벤토리, 포트맵
- `배치`
  - 물리배치
- `업체`
  - 프로젝트 업체, 담당자 연결
- `이력`
  - 감사 이력, 변경 추적

#### 공통 관리 메뉴

- `거래처 관리`
- `카탈로그 / 기준정보`
- `로그`
- `시스템 관리`

### 적용 방향

- `base.html`의 인프라 서브메뉴는 `프로젝트 / 자산 / 네트워크 / 배치 / 업체 / 이력`으로 재구성한다.
- `IP 인벤토리`, `포트맵`, `물리배치`는 자산 내부 탭에만 두지 말고 전역 메뉴에서 바로 진입 가능해야 한다.
- 공통 관리 메뉴는 시각적으로 더 분리된 섹션으로 다룬다.
  - 예: 구분선 강화
  - 예: `기준정보/관리` 라벨 추가
  - 예: 상단 유틸리티 영역 이동

## 제안 2. 화면별 컨텍스트 노출 강화

### 공통 원칙

- topbar 선택기가 있더라도, 화면 본문 상단에 현재 컨텍스트를 다시 보여준다.
- 최소 노출 정보:
  - 현재 고객사
  - 현재 프로젝트
  - 현재 범위 필터
- 컨텍스트 미선택 상태는 빈 화면 대신 "무엇을 먼저 선택해야 하는지"와 바로가기 액션을 제공한다.

### 우선 적용 대상

#### `/contacts`

- 현재 문제:
  - 본문 진입 시 `고객사를 선택하세요.`만 보인다.
- 개선:
  - 화면 설명에서 이 화면이 "프로젝트 참여 업체 + 담당자 + 담당 자산/운영 범위"를 관리하는 곳임을 명시
  - 업체 카드에 `담당 자산` 요약을 노출
  - 현재 구현은 담당 자산 클릭 시 `/assets?asset_id=...`로 이동해 해당 자산 상세 패널을 바로 여는 수준으로 제한
  - 향후 확장 후보:
    - 업체 기준 임시 자산 필터
    - 제조사/제품군/관계유형별 묶음 보기
    - 협력업체 화면에서 자산 상세 사이드패널 직접 호출

#### `/assets`

- 현재 문제:
  - 기능은 많지만, 현재 고객사/프로젝트 기준이 본문에서 약하게 드러난다.
- 개선:
  - 탭 위 또는 필터 바 위에 컨텍스트 배지 추가
  - 현재 필터/선택 범위를 한 줄로 요약
  - Import, 등록, 범위 필터의 관계를 더 명확히 설명

#### `/periods`

- 현재 문제:
  - `프로젝트 목록`이라는 제목만으로는 회계의 `사업 관리`와 역할 구분이 약하다.
- 개선:
  - 보조 설명 추가
  - 예: `고객사별 인프라 수행 프로젝트와 단계, 산출물을 관리합니다.`

## 제안 3. 인프라 화면의 작업 밀도 재정렬

### 유지할 것

- AG Grid 중심의 테이블-밀도 우선 구조
- 좌우 패널 기반 상세 진입 패턴
- Import / 등록 / 필터의 상단 액션 구조

### 바꿀 것

- 탭 수가 많은 화면은 "전역 메뉴 + 화면 내부 보조 탭"으로 역할을 나눈다.
- 필터 바는 최소 검색/상태만 두는 대신, 화면 목적에 맞는 핵심 필터를 추가한다.
  - 예: 프로젝트 범위, 유형, 역할, 운영 상태
- 페이지 헤더에는 "이 화면에서 하는 일"이 드러나는 보조 설명을 넣는다.

## 제안 4. 우선순위

### Phase A. 메뉴 재구성

- `base.html` 인프라 메뉴 재편
- 공통 관리 메뉴 시각 분리
- active 링크 매핑 재검토

### Phase B. 컨텍스트 노출

- `/contacts` 본문 상태 카드 추가
- `/assets` 컨텍스트 요약 추가
- `/periods` 설명 문구 보강

### Phase C. 화면별 세부 개선

- 자산 / 네트워크 / 배치 화면의 헤더, 필터, 빈 상태 메시지 정리
- 미선택/빈 상태 UX 표준화
- 협력업체 화면의 담당 자산 탐색 UX 확장
  - 현재는 자산 상세 점프만 제공
  - 향후 필터/그룹핑/사이드패널 연계는 별도 작업으로 분리

## 완료 기준

- 인프라 사용자가 `IP 인벤토리`, `포트맵`, `물리배치`에 전역 메뉴에서 한 번에 진입할 수 있다.
- `업체`, `자산` 화면에서 현재 고객사/프로젝트 컨텍스트를 본문 상단에서 즉시 파악할 수 있다.
- 공통 관리 화면이 핵심 작업 메뉴와 같은 정보 위계로 보이지 않는다.
- 회계/인프라 모두 "현재 컨텍스트 -> 핵심 액션 -> 결과 그리드" 구조가 더 일관되게 드러난다.

## 구현 시 확인 포인트

- 메뉴 추가가 모바일 폭에서 과밀해지지 않는지 확인
- `active` 상태 매핑이 `/ip-inventory`, `/port-maps`, `/physical-layout`, `/asset-roles`까지 일관되게 동작하는지 확인
- topbar 선택기와 본문 컨텍스트 카드가 서로 다른 상태를 보여주지 않는지 확인
- 권한에 따라 숨겨지는 메뉴가 빈 구획만 남기지 않는지 확인
