# Project Context

## 목적

영업부서의 매출/매입/GP/미수금 관리(회계모듈)와 SI 프로젝트 기술 인벤토리 관리(인프라모듈)를
하나의 웹 애플리케이션에서 통합 운영하는 사내 전용 플랫폼.
ERP 접근 권한이 없는 영업 조직과, 프로젝트 현장의 기술팀을 위해 엑셀 기반 관리를 대체한다.

## 사용자

1. **영업 담당자** (~20명) — 자기 사업의 매출/매입/입금 데이터 입력 및 관리
2. **기술 담당자** (~10명) — 프로젝트 자산/IP/포트맵/정책 관리, 현장 인벤토리 운영
3. **관리 담당자** (1~2명) — 전체 사업 조회, 엑셀 Import, 사용자/역할 관리

## 핵심 문제

- ERP 접근 권한 없이 영업 데이터를 관리해야 함
- 엑셀 기반 관리로 인한 데이터 정합성 문제
- 미수금 현황 파악이 어려움
- 경영진에게 보고할 데이터 집계가 수작업
- 프로젝트 현장의 자산/네트워크 인벤토리를 체계적으로 관리할 도구 부재

## 핵심 데이터

### 회계모듈

- **사업 (Contract)** — 수주 건 단위, 연도별 기간(ContractPeriod)으로 분리
- **Forecast** — 월별 예상 매출/매입 계획
- **Actual** — 월별 실제 매출/매입 실적 (업체별)
- **Receipt** — 입금 내역, ReceiptMatch로 매출에 배분

### 인프라모듈

- **프로젝트 (Project)** — 고객사에 종속된 기술 프로젝트
- **자산 (Asset)** — 서버/네트워크/보안 장비, 프로젝트와 N:M 연결
- **IP 대역 (IpSubnet)** / **포트맵 (PortMap)** — 네트워크 인벤토리
- **정책 (PolicyDefinition/PolicyAssignment)** — 보안 정책 준수 현황

### 공통

- **업체 (Partner)** — 매출처/매입처/고객사 통합 관리

## 핵심 원칙

- 귀속월 기준 매출 관리 (세금계산서 발행월과 다를 수 있음)
- 공급가액 기준 금액 (VAT 별도, 정수 저장)
- 업체 중심 인프라 구조 (자산/IP/포트맵 모두 partner_id 귀속)
- 엑셀 Import 지원 (대량 입력 필요)
- UI는 엑셀과 유사한 테이블 중심 (데이터 밀도 우선)
- 사내 네트워크 전용 (외부 접근 차단)

## 기술 스택

- Backend: Python 3.11+ / FastAPI / SQLAlchemy 2.0 / PostgreSQL 16
- Frontend: Jinja2 템플릿 / AG Grid Community / HTMX / Vanilla JS
- 인증: 세션 기반, RBAC (역할별 모듈 접근 수준 제어)
