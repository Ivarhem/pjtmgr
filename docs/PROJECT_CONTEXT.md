# Project Context

## 목적

영업부서의 매출/매입/GP/미수금을 관리하는 사내 전용 웹 앱.
ERP 접근 권한이 없는 영업 조직을 위해, 엑셀 기반 관리를 대체한다.

## 사용자

1. **영업 담당자** (~20명) — 자기 사업의 매출/매입/입금 데이터 입력 및 관리
2. **관리 담당자** (1~2명) — 전체 사업 조회, 엑셀 Import, 사용자 관리

## 핵심 문제

- ERP 접근 권한 없이 영업 데이터를 관리해야 함
- 엑셀 기반 관리로 인한 데이터 정합성 문제
- 미수금 현황 파악이 어려움
- 경영진에게 보고할 데이터 집계가 수작업

## 핵심 데이터

- **사업 (Contract)** — 수주 건 단위, 연도별 기간(ContractPeriod)으로 분리
- **Forecast** — 월별 예상 매출/매입 계획
- **Actual** — 월별 실제 매출/매입 실적 (거래처별)
- **Receipt** — 입금 내역
- **거래처 (Customer)** — 매출처/매입처 통합 관리

## 핵심 원칙

- 귀속월 기준 매출 관리 (세금계산서 발행월과 다를 수 있음)
- 공급가액 기준 금액 (VAT 별도, 정수 저장)
- 엑셀 Import 지원 (대량 입력 필요)
- UI는 엑셀과 유사한 테이블 중심 (데이터 밀도 우선)
- 사내 네트워크 전용 (외부 접근 차단)

## 기술 스택

- Backend: Python 3.11+ / FastAPI / SQLAlchemy / SQLite
- Frontend: Jinja2 템플릿 / AG Grid Community / HTMX / Vanilla JS
- 인증: 세션 기반, 역할(admin/user) 2단계

## 현재 단계

v0.3 — 파일럿 테스트 진행 중.
핵심 CRUD, Excel Import/Export, 대시보드/보고서, 권한 구현 완료.
감사 로그 연동과 조회 기능은 미완료.

## 향후 방향

감사 로그 → 알림 → 권한 확장 → API 통합 테스트 확대
