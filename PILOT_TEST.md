# 파일럿 테스트 계획

---

## 1. 목적

영업관리 시스템의 핵심 기능을 실제 업무 데이터로 검증하고,
사용자 피드백을 수집하여 정식 운영 전 개선 사항을 도출한다.

### 기대 효과

- 기존 Excel 기반 업무 → 웹 기반 전환 가능성 확인
- 데이터 입력/조회 흐름의 실용성 검증
- UI/UX 개선 포인트 발굴
- 추가 필요 기능 도출

---

## 2. 테스트 대상 기능

### 2.1 사업 관리

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 사업 등록 | 신규 사업 생성 (구분, 담당자, END고객 등) | 필수 항목 누락 시 에러 메시지 |
| 사업 수정 | 기본정보 변경 (담당자, 진행단계 등) | 변경 후 목록 반영 여부 |
| 사업 조회 | 원장 목록 필터 (연도, 구분, 진행단계 등) | 필터 조합 정확성 |
| 내 사업 | 로그인 사용자 기준 자동 필터 | 본인 담당 사업만 표시 |
| 사업기간 관리 | Period 추가/삭제, 탭 전환 | 탭별 데이터 분리 |
| 검수일/발행일 설정 | Deal에 발행일 규칙 입력 | 설정값 저장 및 조회 |

### 2.2 Forecast

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| Forecast 입력 | 월별 예상 매출/GP 입력 | 저장 후 값 유지 |
| Forecast 수정 | 기존 값 변경 후 저장 | 변경값 반영 |
| Forecast → Actual 동기화 | Forecast 가져오기 실행 | 매출 행 자동 생성, 발행일 자동 계산 |

### 2.3 매출/매입 원장

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 행 추가 | 매출/매입 행 수동 추가 | 구분, 거래처, 금액, 발행일 입력 |
| 반복행 생성 | 월 범위 지정 자동 생성 | 지정 기간만큼 행 생성 |
| 행 복제 | 기존 월 데이터 → 다른 월로 복제 | 대상 월 모든 행 복제 |
| 일괄 확정 | 체크박스 선택 후 확정 | 거래처/발행일 누락 시 경고, 상태 변경 |
| 일괄 삭제 | 체크박스 선택 후 삭제 (관리자) | 삭제 확인, 비관리자 차단 |
| 필터 | 귀속월/구분/거래처 필터 | 필터 조합 정확성 |
| 저장 | 변경 사항 일괄 저장 | 신규/수정 행 모두 저장 |

### 2.4 입금 내역

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 입금 등록 | 입금일, 거래처, 금액 입력 | 저장 후 GP 요약 반영 |
| 입금 삭제 | 체크박스 선택 후 삭제 (관리자) | 삭제 후 미수금 재계산 |

### 2.5 거래처 관리

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 거래처 등록 | 신규 거래처 생성 | 중복 이름 차단 |
| 담당자 관리 | 거래처별 담당자 추가/수정/삭제 | 영업/세금계산서 구분 |
| 관련 사업 조회 | 거래처 클릭 → 관련 사업 표시 | 연결 사업 목록 정확성 |

### 2.6 사용자/권한

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 로그인/로그아웃 | 계정 인증 | 잘못된 비밀번호 에러 메시지 |
| 비밀번호 변경 | 최초 로그인 시 변경 강제 | 변경 후 정상 접속 |
| 권한 제어 | 관리자/일반 사용자 기능 차이 | 삭제/Import 관리자만 가능 |
| 데이터 범위 | 일반 사용자 본인 담당만 조회 | 타인 사업 비노출 |

### 2.7 Excel Import

| 시나리오 | 설명 | 확인 항목 |
| --- | --- | --- |
| 사업 Import | 영업기회 시트 업로드 | 사업/기간 자동 생성 |
| Forecast Import | 월별계획 시트 업로드 | Forecast 데이터 반영 |
| Actual Import | 실적 시트 업로드 | 매출/매입 행 생성 |
| 오류 처리 | 잘못된 형식 업로드 | 오류 행 번호/메시지 표시 |

## 3. 테스트 제외 기능

다음 기능은 현재 미구현 상태이며 파일럿 테스트 범위에 포함하지 않는다.

| 기능 | 현재 상태 | 비고 |
| --- | --- | --- |
| 보고서 | 화면 UI만 존재 | 파일럿 이후 개발 |

| 권한 세분화 (manager/viewer) | 미구현 | admin/user만 지원 |
| 발행일 휴일 조정 | 미구현 | 공휴일 달력 미적용 |

## 4. 테스트 시나리오

### 시나리오 A: 신규 사업 등록 → 실적 입력 → 입금 확인

```
1. 사업 관리 → 신규 사업 등록
   - 구분: MA, 사업명 입력, END고객 선택, 담당자 지정
2. 사업기간 추가 (Y26)
   - 진행단계: 50%
3. Forecast 입력
   - 1월~12월 예상 매출/GP 입력 → 저장
4. Forecast 가져오기
   - 매출 행 자동 생성 확인
   - 발행일 자동 입력 확인 (발행일 규칙 설정 시)
5. 매출/매입 원장에서 매입 행 추가
   - 거래처, 금액, 발행일 입력 → 저장
6. 일괄 확정
   - 거래처/발행일 입력된 행 선택 → 확정
7. 입금 등록
   - 입금일, 금액 입력 → 저장
8. GP 요약 확인
   - 매출/매입/GP/GP%/입금/미수 숫자 확인
```

### 시나리오 B: Excel Import → 데이터 검증

```
1. 관리자 계정 로그인
2. 시스템 설정 → Excel Import
   - 사업 Import 실행
   - Forecast Import 실행
   - Actual Import 실행
3. 사업 관리에서 Import된 데이터 확인
4. 사업 상세 → Forecast/원장 데이터 일치 여부 확인
```

### 시나리오 C: 권한 검증

```
1. 관리자 계정: 사업 삭제, 실적 삭제, Import 가능 확인
2. 일반 사용자 계정: 위 기능 차단 확인
3. 일반 사용자: 본인 담당 사업만 조회 확인
4. 일반 사용자: 타인 사업 URL 직접 접근 차단 확인
```

## 5. 피드백 수집 항목

테스트 완료 후 다음 항목에 대해 피드백을 수집한다.

### 기능 관련

- [ ] 필요하지만 빠진 기능이 있는가?
- [ ] 불필요하거나 혼란을 주는 기능이 있는가?
- [ ] 기존 Excel 대비 불편한 점이 있는가?
- [ ] 데이터 입력 흐름이 자연스러운가?

### UI/UX 관련

- [ ] 화면 레이아웃이 업무에 적합한가?
- [ ] 그리드 조작(입력/복사/붙여넣기)이 편리한가?
- [ ] 필터/검색 기능이 충분한가?
- [ ] 에러 메시지가 이해하기 쉬운가?
- [ ] 버튼/메뉴 위치가 직관적인가?

### 데이터 관련

- [ ] 표시되는 숫자(금액, GP, 미수금)가 정확한가?
- [ ] 필요한 필터/정렬 옵션이 있는가?
- [ ] Excel Import 시 데이터가 정확히 반영되는가?

### 성능 관련

- [ ] 페이지 로딩 속도가 적절한가?
- [ ] 대량 데이터 입력 시 응답이 느려지는가?
